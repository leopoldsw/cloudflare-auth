# Security Model

| Threat                                   | Mitigation                                                                        | Evidence                                                                                                                                                         |
| ---------------------------------------- | --------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Account enumeration                      | Generic request responses, deferred email side effects, and dummy token-HMAC work | [routes tests](../tests/routes.test.ts), [email tests](../tests/email.test.ts), [worker runtime](../packages/worker/src/index.ts)                                |
| Credential stuffing                      | D1 rate limits by derived IP and identifier keys                                  | [route tests](../tests/routes.test.ts), [crypto tests](../tests/crypto-password-session.test.ts)                                                                 |
| Brute-force login                        | Password login rate limits and dummy verification path                            | [route tests](../tests/routes.test.ts), [crypto tests](../tests/crypto-password-session.test.ts)                                                                 |
| Reset email abuse                        | Request limits and generic response                                               | [routes tests](../tests/routes.test.ts)                                                                                                                          |
| Magic-link abuse                         | Request and consume limits                                                        | [routes tests](../tests/routes.test.ts)                                                                                                                          |
| Email link scanners                      | `GET` renders confirmation only, `POST` consumes                                  | [routes tests](../tests/routes.test.ts)                                                                                                                          |
| Token replay                             | Single-use token rows with `used_at` and `consume_id`                             | [routes tests](../tests/routes.test.ts), [repository tests](../tests/repositories.test.ts)                                                                       |
| Token leakage                            | Redaction utility for logs, errors, reports, and failure events                   | [security hardening tests](../tests/security-hardening.test.ts), [email tests](../tests/email.test.ts), [CLI tests](../tests/cli.test.ts)                        |
| Open redirects                           | Validate and store redirects at token creation                                    | [routes tests](../tests/routes.test.ts)                                                                                                                          |
| CSRF                                     | Unsafe method origin validation                                                   | [routes tests](../tests/routes.test.ts)                                                                                                                          |
| Session theft                            | HTTP-only opaque cookies and HMAC-hashed token storage                            | [crypto tests](../tests/crypto-password-session.test.ts), [repository tests](../tests/repositories.test.ts)                                                      |
| Session fixation                         | Runtime-generated session tokens only                                             | [routes tests](../tests/routes.test.ts), [worker runtime](../packages/worker/src/index.ts)                                                                       |
| D1 consistency/concurrency               | Guarded token consume predicates                                                  | [repository tests](../tests/repositories.test.ts), [routes tests](../tests/routes.test.ts)                                                                       |
| Email delivery failure                   | Redacted `email_send_failed` auth events and generic request responses            | [email tests](../tests/email.test.ts), [troubleshooting](troubleshooting.md)                                                                                     |
| Secret rotation                          | Current plus previous key-ring parser                                             | [crypto tests](../tests/crypto-password-session.test.ts)                                                                                                         |
| Permissive CORS middleware               | Auth routes own CORS responses                                                    | [routes tests](../tests/routes.test.ts)                                                                                                                          |
| Raw PII in logs, rate limits, and events | HMAC-derived keys and redacted metadata                                           | [crypto tests](../tests/crypto-password-session.test.ts), [security hardening tests](../tests/security-hardening.test.ts), [email tests](../tests/email.test.ts) |
| Bot pressure                             | Optional Turnstile checks before account-specific branching                       | [security hardening tests](../tests/security-hardening.test.ts), [Turnstile docs](turnstile.md)                                                                  |
| Edge floods                              | Optional Cloudflare rate-limit binding before D1 counters                         | [security hardening tests](../tests/security-hardening.test.ts), [rate limiting docs](rate-limiting.md)                                                          |

Known residual risks:

- D1 is the source of truth for rate limits; Cloudflare's rate-limit binding is a coarse prefilter and should not be treated as an account lockout mechanism.
- Password reset confirmation validates the token before hashing the replacement password, but password update, token consume, and session revocation are separate repository operations.
- Hono's no-argument `requireUser()` helper stores the latest mounted config in module scope. Pass the config explicitly when hosting multiple auth configurations in one isolate.
